# Stack CLI V1

## Principles

- branches are the stack unit
- local metadata is the source of truth for stack intent
- remote branches and PRs stay normal and legible
- history rewrites always preview and confirm
- merge queue handoff is single-step and operator-driven
- no GraphQL, no backend, no synthetic refs

## State

- shared graph state: `$(git rev-parse --git-common-dir)/stack/state.json`
- per-worktree op journal: `$(git rev-parse --git-dir)/stack/op.json`

## Implemented Alpha Commands

- `stack move`
- `stack queue`

## Current Guardrails

- write operations refuse to run when repo-level metadata has hard errors
- restack refuses invalid anchors and does not guess a merge base
- interrupted restacks are recorded per worktree and validated before resume
- submit fetches first, previews intended changes, and uses explicit lease-protected pushes
- queue handoff refuses stale remote heads, draft PRs, and non-trunk PR bases
- queue handoff uses `gh pr merge --auto --merge --match-head-commit <sha>` so non-interactive `gh` runs stay valid
- queue handoff supports explicit `merge`, `squash`, and `rebase` strategies
- `stack continue` runs rebases without requiring an interactive editor
- `stack move` does not persist a new parent until the rewrite actually succeeds

## Current Gaps

- merged-parent repair logic is conservative and only auto-applies clean cases
- deleted-branch and empty-field GitHub edge cases for merged-parent sync still need broader live verification
- the TUI is read-only and focused on inspection, not editing
